› Key Principles

There are a number of key principles and motivations that encourage the use of Contextual Design in the design phase.

  • A great system design can not be achieved without having deep understanding of users’ work practice

    • The term "work practice" is defined by Karen Holtzblatt and Hugh Beye as "the complex and detailed set of behaviors, attitudes, goals and intents that characterize a set of users in a particular environment." Therefore, in order to create a system that is accepted and valued by its intended users, it has to be designed to support their work practice.

  • Users can’t convey their work practice articulately to developers.

    Users can’t convey work practice articulately to developers due to two main reasons. First, work practice is something that is experienced rather than said. For example, when you ask a user what do you do when you first set on your desk, he or she may not mention every single detail because some of the things that users do are done unconsciously. However, when you go into their environment and start observing how they do work, and start asking them questions, they will start to be a ware of these important details that would make a great difference in system design. The second reason is that user’s domain is usually complex. This complexity may cause a lot of communication problems between users and developers. This complexity will be significantly reduced when developers are in user’s site communicating and talking to users in their environment, which will facilitate understanding user’s domain.
